# Career Mapping

Knowledge module for understanding career ladders, title normalization, progression timelines, and skill requirements by level.

## Career Ladder Patterns by Industry

### Tech/Software

**Individual Contributor Track:**
```
Level 1-2: Junior/Associate (0-2 years)
  - Learning fundamentals
  - Close supervision
  - Small, well-defined tasks
  - Focus on craft development

Level 3: Software Engineer (2-4 years)
  - Independent contributor
  - Owns features
  - Basic design decisions
  - Debugging complex issues

Level 4: Senior (4-7 years)
  - Leads projects
  - Mentors juniors
  - Designs systems
  - Influences team direction
  - Recognized expertise

Level 5: Staff (7-12 years)
  - Multi-team impact
  - Sets technical direction
  - Solves ambiguous problems
  - Organization-wide influence
  - Technical leadership

Level 6: Principal (10-15 years)
  - Company-wide impact
  - Industry recognition
  - Defines best practices
  - Strategic technical decisions

Level 7+: Distinguished/Fellow
  - Industry-wide impact
  - Creates new paradigms
  - Represents company externally
  - Shapes technology direction
```

**Management Track:**
```
Engineering Manager (EM)
  - Manages 5-10 engineers
  - Performance management
  - Hiring and onboarding
  - Team delivery
  - 1:1s and coaching

Senior EM / Manager of Managers
  - Manages 2-4 EMs
  - Organization design
  - Cross-team coordination
  - Strategic planning

Director
  - Manages department/area
  - Budget responsibility
  - Executive communication
  - Multi-year planning

VP of Engineering
  - Manages multiple directors
  - Company strategy
  - External representation
  - Executive team member

CTO
  - Technical vision
  - Board-level responsibility
  - External facing
  - Technology strategy
```

### Finance/Fintech

**Technical Roles:**
```
Analyst/Associate Developer → Developer → Senior Developer
  → VP (Developer) → Director/SVP → Managing Director

Note: "VP" in finance often = Senior SWE in tech
```

**Quantitative:**
```
Junior Quant → Quant → Senior Quant → Head of Quant
Research Analyst → Senior Analyst → Lead Analyst
```

### Consulting

```
Analyst (0-2) → Consultant (2-4) → Senior Consultant (4-6)
  → Manager (6-10) → Senior Manager → Director/Principal
  → Partner/Managing Director
```

### Startups

```
Early-Stage: Titles matter less
  - Engineer → Senior Engineer → Lead/Staff → VP/CTO

Growth-Stage: Introducing levels
  - Formalizing career ladders
  - Mapping to market levels

Late-Stage: Similar to big tech
  - Full ladder implemented
  - Level matching important
```

### Common Title Variations

**Same Level, Different Titles:**
```
Senior Software Engineer =
  Sr. Developer
  Software Engineer III
  Senior Programmer
  Senior Member of Technical Staff

Staff Engineer =
  Principal Engineer (some companies)
  Software Engineer IV
  Technical Lead (some companies)
  Architect (some companies)
```

### Red Flags in Titles

**Inflated Titles:**
- "Senior" at 2 years experience
- "Architect" for implementation role
- "Lead" with no reports
- "Principal" at small startup

**Deflated Titles:**
- Big tech using lower titles
- FAANG "Senior" = Other company "Staff"
- Consulting firm titles don't map directly

## Progression Timeline Expectations

### Typical Progression

```
Junior to Mid: 2-3 years
  - Demonstrated competence
  - Consistent delivery
  - Growing independence

Mid to Senior: 2-4 years
  - Project leadership
  - Technical depth
  - Mentoring ability

Senior to Staff: 3-5 years
  - Multi-team impact
  - Technical vision
  - Organizational influence

Staff to Principal: 3-5+ years
  - Company-wide impact
  - Industry recognition
  - Strategic leadership
```

### Factors Affecting Progression

**Accelerators:**
- High-impact projects
- Visible contributions
- Strong sponsorship
- Fast-growing company
- Adjacent role experience

**Decelerators:**
- Company politics
- Lack of opportunities
- Missing key skills
- Poor performance
- Market conditions

### Promotion Readiness Signals

**Ready for Senior:**
- Leads projects independently
- Mentors effectively
- Makes sound technical decisions
- Respected by team
- Consistent delivery

**Ready for Staff:**
- Influences beyond team
- Solves ambiguous problems
- Recognized technical authority
- Coaches seniors
- Shapes technical direction

## Skill Requirements by Level

### Technical Skills

**Junior:**
- Core language proficiency
- Basic data structures/algorithms
- Version control
- Debugging skills
- Testing fundamentals

**Mid:**
- Multiple languages/frameworks
- System design basics
- Performance optimization
- API design
- Database proficiency

**Senior:**
- Deep expertise in areas
- Complex system design
- Technical decision making
- Code review excellence
- Production operations

**Staff:**
- Architectural thinking
- Cross-system expertise
- Technical vision
- Technology evaluation
- Innovation leadership

### Leadership Skills

**Junior:**
- Receiving feedback
- Asking questions
- Pair programming

**Mid:**
- Self-direction
- Code reviews
- Documentation
- Knowledge sharing

**Senior:**
- Mentoring
- Project leadership
- Technical decisions
- Process improvement

**Staff:**
- Team influence
- Cross-team coordination
- Technical strategy
- Organizational change

### Business Skills

**Junior:**

- Basic estimation

**Mid:**
- Stakeholder communication
- Trade-off analysis
- Task prioritization

**Senior:**
- Product awareness
- Business context
- Resource planning
- Risk assessment

**Staff:**

- Business case development

## Career Transition Patterns

### Common Lateral Moves

**Engineering to:**
- Product Management (technical PM)
- Technical Program Management
- Developer Relations
- Solutions Architecture
- Engineering Management

**Between Domains:**
- Backend ↔ Frontend → Fullstack
- Backend → Platform/Infrastructure
- Any → Data Engineering
- Mobile ↔ Frontend

### Career Change Considerations

**Easier Transitions:**
- Within same company
- Leveraging transferable skills
- With bridge roles
- During job changes

**Harder Transitions:**
- Across industries
- Dropping seniority level
- Without relevant experience
- In tight job markets
